### 走进函数式

```
    原因：作为开发人员，喜欢那些拥有简洁应用结构并可以增强软件扩展性的框架
         今天已经可以实现很多以前技术上不可行的东西。
         Node.js来编写大型服务器端应用程序，
         大量的业务逻辑放到客户端去实现，是的服务端非常轻巧
         这些就需要与外部存储的交互、创建异步进程、处理事件等等

```

```

    需求：面向对象设计 有助于解决一部分问题，
        但是由于JavaScript是一种拥有很多共享状态的动态语言，
        用不了多久，代码就会积累足够的复杂性，变得笨拙而难以维护

        面向对象的设计的确能够一定程度地缓解这个问题

```

```
    原则：需要遵从一下原则

        - 可扩展性
        - 易模块化
        - 可重用性
        - 可测性
        - 易推理性
```

```
    解决：
        函数式编程就是你需要的编程范式
        不是一种新工具或新的API
        而是另一种解决问题的方式，一旦你了解了它的基本原则，
        所要解决的问题将变得很直观
```

- 1.1函数式编程有用吗

    ```
        函数式编程很重要，大多数主流编程语言都提供原生的或基于API的函数式支持
        在JavaScript的上下文中，函数式思想可以用来塑造令人难以置信的语言特性，
        帮助你编写干净的、模块化的、可测试的并且简洁的代码
        使得开发过程中更加高效
    ```

    ```
        JavaScript缺乏一些能够妥当管理状态的原生结构--这种动态语言将管理状态的职责交给了开发人员
        这个问题不会影响规模较小的脚本代码，
        但是代码量不断增长，会变得越来越难以控制

        而编写函数式的JavaScript代码能够克服以上提到的大部分问题
        可以编写出易于推理和理解的代码
        函数式的JavaScript不仅能够提高整个应用程序的质量
        也能够更好地了解并精通JavaScript语言本身
    ```

    ```
        因为函数式编程是一种编写代码的方式，
        不是框架或工具
        函数式的思维方式与面向对象的思维方式完全不同
    ```
